# V1.1 内网部署说明

更新时间：2026-07-14

本说明用于把 `v1.1-deployment-ready` 部署到公司内网 Windows 机器，供设计师试用。当前推荐部署方式是：一台固定电脑或服务器同时运行前端、后端和 DWG 转换器，设计师通过局域网访问前端页面。

## 1. 部署目标

- 前端 Web：`http://部署机器内网IP:3010/`
- 后端 API：`http://部署机器内网IP:8010/`
- 设计师只需要访问前端地址。
- 上传 DXF 会直接解析；上传 DWG 会由后端调用 ODA File Converter 转成临时 DXF 后解析。

## 2. 机器准备

建议部署目录：

```powershell
D:\cad-budget-program-G
```

部署机器需要安装：

- Git
- Node.js 22 LTS 或当前项目可用版本
- Python 3.13/3.14
- ODA File Converter，建议路径：

```powershell
D:\ODA\ODAFileConverter\ODAFileConverter.exe
```

如果 ODA 安装在其它路径，注册自启动任务时需要传入 `-OdaFileConverterPath`。

## 3. 首次部署

以管理员身份打开 PowerShell。

```powershell
cd D:\
git clone https://github.com/giwawe/G-cad-budget-program.git cad-budget-program-G
cd D:\cad-budget-program-G
git checkout v1.1-deployment-ready
```

安装后端依赖：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r server\requirements.txt
```

安装前端依赖并构建：

```powershell
npm install
node node_modules\next\dist\bin\next build apps\web
```

## 4. 手动启动验证

启动后端：

```powershell
.\deploy\start-api.ps1
```

启动前端：

```powershell
.\deploy\start-web.ps1
```

执行健康检查：

```powershell
.\deploy\health-check.ps1
```

如果部署机器本机检查通过，再到设计师电脑访问：

```text
http://部署机器内网IP:3010/
```

上传一张 DXF 和一张 DWG，确认能解析、查看预算并下载预算表。

## 5. 注册开机自启动

确认手动启动正常后，继续以管理员身份运行：

```powershell
.\deploy\register-scheduled-tasks.ps1 -CreateFirewallRules
```

默认会注册两个任务计划程序：

- `CAD Budget API`
- `CAD Budget Web`

默认运行身份为 `SYSTEM`，触发方式为开机启动。也就是说部署机器重启后，即使没有人登录，也会自动启动后端和前端。

如果 ODA File Converter 不在默认路径，使用：

```powershell
.\deploy\register-scheduled-tasks.ps1 `
  -OdaFileConverterPath "D:\你的ODA路径\ODAFileConverter.exe" `
  -CreateFirewallRules
```

注册完成后可以立即启动任务：

```powershell
Start-ScheduledTask -TaskName "CAD Budget API"
Start-ScheduledTask -TaskName "CAD Budget Web"
```

再执行：

```powershell
.\deploy\health-check.ps1
```

## 6. 日志位置

脚本会把运行日志写到：

```text
D:\cad-budget-program-G\logs
```

常用日志：

- `api-8010.out.log`
- `api-8010.err.log`
- `web-3010.out.log`
- `web-3010.err.log`
- `web-build.out.log`
- `web-build.err.log`

如果页面打不开，先看 `web-3010.err.log`；如果上传解析失败，先看 `api-8010.err.log`。

## 7. 停止服务

手动停止当前监听 `3010` 和 `8010` 的前后端进程：

```powershell
.\deploy\stop-all.ps1
```

如果是任务计划程序托管，停止后任务不会自动立即重启；下次开机或手动 `Start-ScheduledTask` 会再次启动。

## 8. 更新到新版本

后续有新版本时，建议按以下顺序更新：

```powershell
cd D:\cad-budget-program-G
.\deploy\stop-all.ps1
git fetch --tags origin
git checkout <新版本标签>
npm install
node node_modules\next\dist\bin\next build apps\web
.\.venv\Scripts\python.exe -m pip install -r server\requirements.txt
Start-ScheduledTask -TaskName "CAD Budget API"
Start-ScheduledTask -TaskName "CAD Budget Web"
.\deploy\health-check.ps1
```

## 9. 常见问题

### 页面打不开

先在部署机器本机打开：

```text
http://127.0.0.1:3010/
```

如果本机可以打开、设计师电脑打不开，通常是防火墙或内网 IP 不对。重新运行：

```powershell
.\deploy\register-scheduled-tasks.ps1 -CreateFirewallRules
```

### 上传 DXF/DWG 提示接口失败

检查后端：

```powershell
Invoke-WebRequest http://127.0.0.1:8010/health -UseBasicParsing
```

如果本机后端正常、设计师电脑失败，确认前端访问地址是内网 IP，而不是 `127.0.0.1`。

### DWG 上传失败

确认 ODA File Converter 路径存在：

```powershell
Test-Path "D:\ODA\ODAFileConverter\ODAFileConverter.exe"
```

如果安装路径不同，重新注册任务并传入实际路径：

```powershell
.\deploy\register-scheduled-tasks.ps1 -OdaFileConverterPath "D:\实际路径\ODAFileConverter.exe"
```

### 样式丢失

前端必须从 `apps\web` 目录以生产模式启动。不要在项目根目录手动运行 `next dev apps\web` 作为试用部署。当前 `deploy\start-web.ps1` 已按正确目录启动。
