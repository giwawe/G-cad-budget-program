# DXF 空间算量验证工具 MVP 需求文档

## 产品目标

验证按规范绘制的商品房 DXF 是否能自动生成准确、可追溯、可校对的空间工程量依据表。

第一期不追求完整报价，不生成最终装修报价单。核心成功标准是：报价员能用系统结果替代大部分手工空间算量。

## 用户

装修公司的报价员。

他们现在从 CAD 平面方案中手动获得空间数据，再填写报价表。主要痛点是报价慢、规则较乱、数据依据不准确。

## 输入

- 按《CAD 报价算量出图规范 v1》绘制并导出的 DXF。
- 项目名称。
- 项目默认层高。
- 默认窗高。
- 默认门高。
- 可选楼层默认层高。

## 输出

空间工程量校对表。

| 字段 | 说明 |
|---|---|
| 楼层 | 来自空间名称、`QUOTE_FLOOR` 或人工校正 |
| 空间名称 | 来自空间内文字 |
| 空间类型 | 根据名称推断，可人工修改 |
| 地面面积 | `QUOTE_ROOM` 闭合边界面积 |
| 顶面面积 | 默认等于地面面积 |
| 墙面计量长度 | 与空间关联的 `QUOTE_WALL` 长度 |
| 层高 | 按层高优先级解析 |
| 窗宽合计 | 空间关联窗户宽度总和 |
| 窗洞面积 | 窗宽 * 窗高 |
| 门宽合计 | 空间关联门洞宽度总和 |
| 墙面展开面积 | 墙面计量长度 * 层高 |
| 乳胶漆面积 | 墙面展开面积 - 窗洞面积，门默认不扣 |
| 计算依据 | 解释公式和参数来源 |
| 异常 | 未闭合、无名称、未归属门窗等 |
| 状态 | 待确认、已确认、需修图 |

## 第一版空间类型

必做：

- 客厅
- 餐厅
- 过道
- 厨房
- 卫生间
- 阳台
- 卧室
- 书房

预留或半自动：

- 楼梯
- 楼梯过道
- 露台
- 外墙

识别但默认排除：

- 电梯井
- 设备井
- 管井

## 算量规则

```text
地面面积 = QUOTE_ROOM 闭合多段线面积
顶面面积 = 地面面积
墙面计量长度 = 空间关联 QUOTE_WALL 长度
墙面展开面积 = 墙面计量长度 * 层高
窗洞面积 = 窗宽合计 * 窗高
乳胶漆面积 = 墙面展开面积 - 窗洞面积
门洞默认不扣墙面面积
```

## 成功指标

| 指标 | 目标 |
|---|---|
| 空间识别率 | 95% 以上空间能正确识别名称和边界 |
| 地面面积误差 | 与人工结果误差 <= 1% |
| 墙面计量长度误差 | 与人工结果误差 <= 1% |
| 窗户归属 | 能正确归属到对应空间 |
| 依据可追溯 | 每个核心数量都有公式说明 |
| 异常可发现 | 不闭合、无名称、未归属门窗必须提示 |

## 非目标

- 不兼容任意历史 DWG。
- 不直接承诺完全自动报价。
- 第一版不做 CAD 插件。
- 第一版不强做楼梯踏步自动算量。
- 第一版不强做外墙洞口自动扣减。
