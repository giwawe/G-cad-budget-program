# V2 后续升级路线清单

更新时间：2026-07-13

本清单用于把 V1 试用版之外的想法收进后续版本，避免影响当前可落地版本的稳定性。V1 继续聚焦规范方案文件上传、工程量校对、健康检查和 Excel 草稿导出。

## V2 核心方向

V2 目标是从“单机工作台式试用工具”升级为“公司内部报价协作系统”。升级时应保持 V1 的确定性算量和真实图纸回归能力，不把平台化功能和算量规则大改混在同一次发布里。

## 优先级 1：项目与客户流程

- 设计师注册和登录。
- 设计师创建客户信息，包括客户、地址、设计师、报价员、报价日期等字段。
- 客户与设计方案绑定，后续重新上传或调整方案时能沿用客户信息。
- 客户方案版本记录，保留每次 DXF、校对快照和报价 Excel 的阶段结果。
- 报价抬头信息从客户档案自动带入 Excel。

## 优先级 2：管理端与全局数据

- 管理层后台登录。
- 设计师账号管理。
- 客户和项目查询。
- 全局报价规则维护，包括主材、辅材、人工三段价格。
- 管理端查看设计师客户方案、校对状态和报价导出记录。
- 报价规则修改应保留版本，避免旧项目被新单价静默影响。

## 优先级 3：DWG 与文件处理深化

- V1.1 已支持 DWG 上传入口，并通过 ODA File Converter 转为临时 DXF 后解析；后续补齐服务器安装、部署和运维说明。
- 保留 DXF 作为稳定解析路径，持续优化 DWG 转换失败时的可操作提示和重试流程。
- 记录 DWG 原始文件、转换后的 DXF 摘要和转换错误，方便管理端追踪。
- 上传文件和校对快照要能关联到客户方案版本。

## 优先级 4：前端体验升级

- 重新整理首页工作流：上传、复核、校对、报价导出四步更清楚。
- 设计师默认只看必要信息：建筑面积、地面面积、墙线、门窗、健康检查、报价风险。
- 工程量校对表默认精简列，复杂字段收进展开详情。
- 方案完整性复核保留给设计师查漏，但默认隐藏洁具、水电推荐点位等高级图层。
- 报价输出模式和报价规则面板优化视觉层级，但不改变业务逻辑。

## 优先级 5：扩展接口

- 合同信息接口预留。
- 收款、施工交底、材料选样等模块预留。
- 报价 Excel 与后续合同/预算审批流程的字段映射预留。
- API 权限和审计日志预留。

## 暂不混入 V1.0.x

- 不做完整后台。
- 不做账号体系。
- 不做 DWG 原生解析重构；DWG 继续走转换为 DXF 的边界。
- 不做大规模前端视觉改版。
- 不改已经通过真实图纸回归的报价口径，除非 V1 试用发现明确金额错误。

## 进入 V2 前的准入条件

- V1 至少完成 1-2 个真实客户项目试用。
- `docs/v1-trial-feedback.md` 中没有未关闭的“必须修”金额或导出问题。
- 真实图纸回归脚本可以稳定跑通 5 张样例图纸。
- V2 第一阶段需求拆成可单独验收的小任务，不一次性重写前后端。
